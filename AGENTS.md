# AGENTS.md

Orientation for AI agents working in this repo. Keep this file up to date in the
same change whenever conventions, structure, or workflows change.

## What this is

My personal dotfiles, applied with a plain shell script (`./setup`). No Ansible,
no nix — just symlinks and a few helpers. Must run on **macOS and Linux** (an
Alpine Docker image runs `./setup`), so keep `bin/` scripts POSIX-portable and
avoid GNU-only/BSD-only flags.

## Layout

- `setup` — main install script (symlinks configs, installs tools).
- `bin/` — personal scripts on `$PATH`. Keep portable.
- `scripts/` — maintenance (`update.sh`, `release.sh`).
- `Brewfile` — auto-regenerated by `scripts/update.sh` via `brew bundle dump`;
  hand-edits don't persist unless the package is actually installed/removed.
- Per-tool config dirs: `ghostty/`, `rio/`, `fish/`, `nvim/`, `tmux/`, `git/`, …

## Conventions

- Conventional Commits with scope (e.g. `fix(rio): …`), one logical change per
  commit, signed off (`-s`).
- Don't commit speculative work; wait for an explicit "commit".
- Integrate upstream branches with **merge, never rebase**.
- Terminal theming: Catppuccin Mocha + Nerd Font glyphs.

## Keep terminal emulator configs in sync

`ghostty/config` and `rio/config.toml` configure two terminal emulators that I
use interchangeably. **They must stay behavior-equivalent.** Whenever you change
one, mirror the change in the other in the same commit:

- **tmux keybindings** must match exactly. Both send the same control sequences
  (tmux prefix is `C-b` = `\x02` / `\u0002`). Ghostty `keybind = cmd+X=text:\x02Y`
  corresponds to rio `{ key = "X", with = "super", esc = "\u0002Y" }`.
- **Fonts**: same family (Berkeley Mono), size, features
  (`zero`, `calt`, `liga`), and Nerd Font symbol/codepoint maps.
- **Theme/appearance**: Catppuccin Mocha, black background, maximized window.

When only one emulator supports a given option, document why in a comment rather
than silently diverging. Before finishing any change to one of these files,
diff the two and confirm the shared settings still match.
